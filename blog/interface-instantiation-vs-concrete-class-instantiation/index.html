Interface Instantiation vs Concrete Class Instantiation

March 24

Interfaces cannot be instantiated by definition. You always will instantiate a concrete class.

So in both statements your instance is actually of the concrete class type.

The difference between instantiating the concrete class via the interfaces, as far as C# is concerned, is that your car is something that implements the interface, which will only have the interface's methods available to it.

This can be demonstrated with the Roadster concrete class IVehicle interface. We will first build these classes, then make a few calls that will give an easily understandable and practical understanding of what we just explained, and how it will change how in this case, the Roadster, will behave.
